第5章 静态方法　　　

　　在第四章的例4-13中，可以将程序按如下方法进行描述：
{
//输入学生的姓名、数学成绩和Java程序设计的成绩
//输入需要查询的学生姓名
//检索数组，显示查找的结果
}
　　这种轮廓描述可以方便用户进行程序的编码，并能使程序结构清晰。但也存在其局限性：
　　 程序中每个任务只能通过注释来说明；
　　对于较大的程序，要在一个main中包含所有的内容，结构性较差；
　　如果程序中需要重复多次完成某一功能，则程序中需要重复多次编写同样的代码。
　　为了便于规划、组织、编程和调试，一般的做法是将一个大的程序分解成若干个程序模块，每一个模块实现一部分功能。这样模块的功能不需要通过注释，而是通过模块名来表示，同时对于重复多次使用的功能通过程序调用而不是重复编码来实现。Java语言中提供两种方法实现模块化：过程（procedure）和函数（function）。
