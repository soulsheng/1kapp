多态性

　　多态性是面向对象程序设计语言中除了数据抽象(类和对象)和继承以外的又一个重要特征，它允许程序员改进程序代码组织，创建易读性、可扩展性的程序。
　　封装是通过结合特性（属性）和行为（方法）来创建新的数据类型（类和对象）
　　隐藏通过细节private来分离实现与接口，即外部不能访问private成员，只能通过公共的接口（方法）访问该对象成员
　　多态性的特点是采用同名方法，根据调用方法时传递参数的类型、数量，执行不同的实现方法。
Java语言的多态性是通过以下机制实现的：
　　向上引用：向上引用机制通过允许父类变量指向其子类对象实现，在程序中可以在方法的形式参数定义父类变量，具体执行是根据调用该方法的实际参数（子类对象）执行不同的代码。其特点就在于“忘记”继承类，以基类作为参数编制一个方法，实现不同继承类的调用。
　　方法联编（Method-call binding）：向上引用涉及面向对象程序设计中联编的概念，联编指的是连接一个方法调用到一个方法体（实现），它分为早期联编和迟后联编，前者是指在程序运行以前完成（编译和连接），后者则是基于对象类型在运行时完成，而在Java语言中除了final定义的方法外，所有方法联编均使用迟后联编。
　　重载：重载分为两种情况：一是在类中创建多个同名方法，其参数数量和类型不同，它使得Java语言中该方法对各种对象都有一个统一接口（方法名）而调用方法的不同是有调用该方法的参数类型和数量决定。二是在类的继承机制中，子类可以继承父类的方法，也可以重载父类的方法（有时也称为覆盖）。

多态性举例

　　【例9-11】迟后联编举例，一个基类Shape 以及各种子类：Circle, Square, Triangle等等。
　　向上引用能在一个语句中出现：Shape s = new Circle()；
　　假设调用父类方法（该方法已被子类重载）：s.draw()；则由于多态性的迟后联编，Circle.draw( ) 被调用。这里父类Shape 对任何其子类建立公共接口，子类重载这些定义来提供其特殊的行为。
